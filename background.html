        # My Involvement in the Maritime Leave Module

        I was enlisted in this project as I had prior experience building the Leave Management module for Rebus Software PS2000 payroll and experience with maritime payrolls at Union Shipping. The project was lacking a clear vision of what to build to meet the client’s requirements, so we decided to proceed in a phased approach.

        # Requirements Phase

        The Project Manager and I worked for two months refining the specification and prototyping in Transact SQL code. We shared the prototype with the client and discussed details during weekly teleconference meetings.

        It became clear that the requirements were best illustrated by example, but this was hampered by language difficulties and a lack of established problem domain lexicon. To assist the process I built the Test Runner application detailed below. This allowed us to build small automated test data sets for each feature.

        At the end of the requirements phase, the Client Services Manager, the Project Manager and myself visited the client in Marseille to agree the contract and iron out final details face to face.

        # Build Phase

        I then proceeded to develop the production version in C#, updating the test runner to work with .Net assemblies. The client continued to validate releases and provide feedback.

        # User Interface Refinement

        After the leave module was deployed to production, we went through period of end-user acceptance where the UI was refined to better fit the user’s perception of the information displayed.
